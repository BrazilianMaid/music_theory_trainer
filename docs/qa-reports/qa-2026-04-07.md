## Blocker Detail

**Edge case: empty question bank produces unhandled crash**

- **Location:** `src/App.jsx`, `QuizWrapper` component
- **Trigger:** `buildSession(questions, topicId)` returns `[]` (topic ID has no matching questions)
- **Current behaviour:** `useQuiz([])` → `currentQuestion = undefined` → `QuizScreen` renders `currentQuestion.question` → TypeError crash
- **Required behaviour per `ui-design.md`:** Show centred message "No questions available for this topic yet." with a "Back to Topics" button
- **Required fix:** Add a session length guard in `QuizWrapper` before rendering `QuizScreen`:

```jsx
function QuizWrapper({ topic, onComplete, onGoBack }) {
  const [sessionQuestions] = useState(() => buildSession(questions, topic.id))
  const quiz = useQuiz(sessionQuestions)

  // Defensive: no questions available for this topic
  if (sessionQuestions.length === 0) {
    return (
      <div className="text-center py-16 px-4">
        <p className="text-gray-600 mb-4">No questions available for this topic yet.</p>
        <button onClick={onGoBack} className="...">Back to Topics</button>
      </div>
    )
  }

  useEffect(() => { ... }, [quiz.isComplete])
  return <QuizScreen quiz={quiz} />
}
```

`onGoBack` (calls `handleChooseTopic`) must be threaded from `App` into `QuizWrapper` and used only for this fallback — not the same as the removed `onChooseTopic` prop, which was removed because it was unused in the normal flow.
